# Coworking with Claude

Most people use Claude as a chatbot: ask a question, get an answer, move on. To really master Claude for work, shift your mindset from **chatting** to **coworking**, treating Claude as a collaborator on ongoing, multi-step work.

## Chat vs. cowork

| Chatting | Coworking |
| --- | --- |
| One question at a time | A persistent workspace with shared context |
| Re-explain yourself each time | Claude remembers the project's background |
| Copy answers out by hand | Edit living **Artifacts** in place |
| Starts from zero | Builds on your files and prior decisions |

## The coworking toolkit

`claude.ai` gives you several features that turn conversations into real collaboration:

- **[Projects](/coworking/projects)**: a dedicated space for a body of work, with its own knowledge and custom instructions that apply to every chat inside it.
- **[Artifacts](/coworking/artifacts)**: documents, code, diagrams, and apps that appear beside the chat and can be edited and iterated on.
- **[Files & long context](/coworking/files)**: upload reports, spreadsheets, and images for Claude to read and work with.
- **[Memory](/coworking/memory)**: Claude remembering useful facts and preferences across conversations.

Beyond `claude.ai`, coworking extends into **[Skills](/skills/overview)** (packaged expertise) and **[integrations](/integrations/mcp)** (connecting your real tools).

## A coworking example

Imagine you run weekly customer-research synthesis:

1. Create a **Project** called "Customer Research."
2. Add **custom instructions**: your format, tone, and what you care about.
3. Upload your **interview guide and personas** as project knowledge.
4. Each week, start a chat, drop in new transcripts, and ask for the synthesis.
5. Claude produces an **Artifact** report you refine in place.

You never re-explain the context. That's coworking, and it compounds over time.

Start with the cornerstone feature: [Projects](/coworking/projects).
